# 路线图

## 第 0 阶段：初始化

- 搭建 MkDocs Material 网站。
- 建立课程双入口：首字母索引、学院入口。
- 建立课程页模板和资料目录模板。
- 配置 GitHub Pages、Git LFS、giscus 预留位。

## 第 1 阶段：种子内容

- 写完新生入门、选课、绩点、保研、出国、就业、科研入门。
- 从维护者熟悉的课程开始建立高质量课程页。
- 收集一批课程资料和链接。
- 建立常用政策入口和校园生活轻量索引。

## 第 2 阶段：公开上线

- 修正站点链接和 GitHub Pages 配置。
- 开启 Discussions 和 giscus。
- 发布第一版 README 和贡献指南。
- 邀请少量同学试用并反馈。

## 第 3 阶段：社区扩展

- 按学院逐步补课程。
- 建立课程资料贡献流程。
- 收集课程评论和课程体验。
- 根据资料规模决定是否使用 Release 或外部存储。

## 第 4 阶段：质量维护

- 定期检查过期政策链接。
- 清理重复课程页和失效资料。
- 对争议评论和下架请求建立处理记录。
- 逐步补充更多学院和路径案例。
